Instructions for setting up Batkeeping

 

After completing these steps, you will be able to run the application.  You will also have a development environment in which you will be able to make changes to the database or to the application and commit your changes to the project at rubyforge.

 

Download:

 

1.      MySQL 5.0 Community Server (Essentials 16.8MB): http://dev.mysql.com/downloads/mysql/5.0.html#downloads

 

2.      MySQL GUI tools: http://dev.mysql.com/downloads/gui-tools/5.0.html

 

3.      Ruby: http://rubyforge.org/frs/?group_id=167

 

4.      RubyGems: http://rubyforge.org/frs/?group_id=126

 

5.      Tortoise CVS: http://www.tortoisecvs.org/download.shtml

 

6.      RMagick: http://rubyforge.org/projects/rmagick

 

While you are waiting for these to finish downloading, you can go to rubyforge.org, create an account, and ask one of the admins add you to the batkeeping project.

 

Installation:

 

1.      Install MySQL Server 5.0

a.      Choose Typical

b.      Skip Sign-Up

c.       Choose configure MySQL Server now

d.      Standard Configuration

e.      Install as a windows service and include bin directory in windows path

f.        create a root password for your mysql server

g.      Click execute!

 

2.      Install MySQL GUI Tools

a.      Default options

 

3.      Install Ruby

a.      Install the default options

b.      You can change the default location if you want (C:\Program Files\ruby)

 

4.      Install RubyGems

a.      Extract RubyGems to your desktop

b.      Start - Run - cmd - OK

c.       "cd desktop\rubygems-0.9.0"

d.      "ruby setup.rb"

e.      leave the command window open for the next task

 

5.      Install Rails

a.      Type in a command window: "gem install rails --include-dependencies"

b.      This will take a while

 

6.      Install Tortoise CVS

a.      Default Installation

b.      Restart your computer

 

7.      Install RMagick

a.      Extract the .zip file.

b.      Run the .exe file (ImageMagick-6.3.0-7-Q8-windows-dll.exe)

c.       Open a command window and "cd" to the unzipped directory (probably called: RMagick-1.14.1_IM-6.3.0-7-Q8), then run the following command: "gem install rmagick-1.14.1-win32.gem"

d.      It should say: "Successfully installed rmagick, version 1.14.1.  If it does, you are good to go.

 

Configuration:

 

1.      First, download the CVS repository (our batkeeping rails files)

a.      go to rubyforge.org, create an account, and have one of the admins add you to the batkeeping project (hopefully you have already done this)

b.      create a folder where the batkeeping folder will be held (My Documents works okay)

c.       right click on the folder, and choose cvs checkout

d.      you should now have a checkout module dialog

e.      fill in the following textboxes:

I.                    Change protocol to Secure shell (:ext:)

II.                  server: "rubyforge.org"

III.                Repository folder: "/var/cvs/animals"

IV.                Module: "batkeeping" --> make sure it's lowercase!!!

f.        click OK, then yes to accept the security alert

g.      Enter your rubyforge.org password at the prompt

h.      Woo Hoo!  You now have the most current CVS version of our batkeeping project.  There should now be a green check mark on it titled "batkeeping".

 

2.      Setup the MySQL database

a.      go to start - programs - MySQL Server 5.0 - MySQL Command Line Client

b.      enter your mysql root password

c.       type "create database batkeeping;" and then "exit"

d.      now open the MySQL Administrator

e.      Click Restore (along the left panel)

f.        Click Open Backup File, then navigate to the newly created batkeeping directory you just got from the CVS download, then into "db", then open the most recent db_data.sql file

g.      Now click Start Restore

h.      You can now close the MySQL Admintrator.  Your database was created!

 

3.      Startup WEBrick

a.      open up a cmd window

b.      cd into the batkeeping directory

c.       type "ruby script/server"

d.      open up your favorite web browser

e.      type in the address bar: "http://localhost:3000/"

f.        you should see the login page for the batkeeping application

 

Editing Files

 

1.      Downloading from CVS

a.      Whenever you begin working on something for the project, make sure you start by doing a CVS Update on the batkeeping direcotry

 

2.      ScITE is a very good text editor.  It does formatting for ruby (unfortunately it doesn't support the rails webpages .rhtml) and it has tabbed viewing.  If you want to try something that supports the rails .rhtml pages, you could try jEdit - it supports .rhtml files after downloading a few additional plugins.

 

3.      Updating to CVS

a.      Whenever you finish working on something for the project, make sure you upload it to the CVS repository, along with a comment about what you did, what’s broken, etc.

b.      If you added any files, you will have to right click on the folder, and choose 'CVS Add Contents' before committing.

c.       In order to upload changes you made you need to go to the batkeeping folder, right click, and choose CVS Commit.

d.      Again, make sure you leave comments on what you worked on.
